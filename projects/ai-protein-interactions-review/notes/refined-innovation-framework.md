## 一、核心概念框架深化

### 1.1 "Mapping vs Prediction" 的理论内涵

**传统视角的局限**:
现有综述将 PPI 预测视为一个分类问题：给定两个蛋白质，预测它们是否相互作用。这种视角忽略了 PPI 研究的真正目标——理解细胞内完整的相互作用网络。

**新视角的理论基础**:

```
┌─────────────────────────────────────────────────────────────────┐
│                    PPI 研究的终极目标                            │
│                                                                 │
│   构建"Interactome"——细胞内完整的蛋白质相互作用图谱              │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Prediction (预测)              Mapping (映射)                 │
│   ─────────────────              ─────────────                  │
│   · 局部问题                     · 全局问题                      │
│   · 二分类任务                   · 网络构建任务                  │
│   · 精度导向                     · 覆盖度导向                    │
│   · 方法评估                     · 网络质量评估                  │
│                                                                 │
│   输入: 蛋白对                   输入: 全蛋白质组                │
│   输出: 相互作用概率             输出: 相互作用网络              │
│   评估: Accuracy, AUC           评估: 网络完整性, 功能模块      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**核心论点**:
> 预测只是映射的工具，但现有研究混淆了手段与目的。高质量的预测方法不一定能构建高质量的相互作用网络。

### 1.2 为什么这个区分重要？

| 问题 | Prediction 视角 | Mapping 视角 |
|------|-----------------|--------------|
| 假阳性如何处理？ | 指标的一部分 | 网络中的噪声边，影响后续分析 |
| 假阴性如何处理？ | 指标的一部分 | 网络不完整，缺失关键连接 |
| 如何选择阈值？ | 最大化F1 | 平衡网络完整性与准确性 |
| 如何评估结果？ | Benchmark对比 | 网络拓扑分析、功能富集 |
| 研究目标是什么？ | 开发更好的分类器 | 构建生物学有意义的网络 |

**学术贡献**: 提供新的概念框架，重新审视整个领域

## 二、方法论框架设计

### 2.1 双轨方法论分类

**创新点**: 不是简单地按算法分类（CNN/RNN/GNN），而是按任务类型分类

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI for PPI: 双轨方法论框架                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Track A: Modeling Methods (建模方法)                           │
│  ══════════════════════════════════                             │
│  目标: 预测特定蛋白质相互作用                                    │
│                                                                 │
│  A1. Interaction Site Prediction (位点预测)                     │
│      └─ 预测蛋白质表面的相互作用残基                            │
│                                                                 │
│  A2. Pair-wise Interaction Prediction (对预测)                  │
│      └─ 预测两个蛋白质是否相互作用                              │
│                                                                 │
│  A3. Binding Affinity Prediction (亲和力预测)                   │
│      └─ 预测相互作用强度或突变效应                              │
│                                                                 │
│  A4. Interface Structure Prediction (界面结构预测)              │
│      └─ 预测复合物的三维结构                                    │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Track B: Mapping Methods (映射方法)                            │
│  ═══════════════════════════════                                 │
│  目标: 构建完整的相互作用网络                                    │
│                                                                 │
│  B1. Network Construction (网络构建)                            │
│      └─ 从蛋白质列表构建相互作用网络                            │
│                                                                 │
│  B2. Network Completion (网络补全)                              │
│      └─ 补充已知网络中缺失的相互作用                            │
│                                                                 │
│  B3. Network Refinement (网络精化)                              │
│      └─ 去除假阳性，提高网络质量                                │
│                                                                 │
│  B4. Cross-species Mapping (跨物种映射)                         │
│      └─ 将一个物种的PPI映射到另一个物种                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 两个轨道的关系

```
                    Modeling Methods
                          │
                          │ 提供预测结果
                          ▼
    ┌─────────────────────────────────────────┐
    │                                         │
    │         Prediction Results              │
    │         (相互作用概率/位点/亲和力)       │
    │                                         │
    └─────────────────────────────────────────┘
                          │
                          │ 整合策略
                          ▼
                    Mapping Methods
                          │
                          │ 构建网络
                          ▼
    ┌─────────────────────────────────────────┐
    │                                         │
    │         Protein Interaction Network     │
    │                                         │
    └─────────────────────────────────────────┘
```

**关键洞察**: 现有综述只关注上半部分（Modeling），忽略了下半部分（Mapping）的挑战。

## 三、方法选择决策框架

### 3.1 决策框架设计原则

**原则1: 问题导向**
- 从研究问题出发，而非从方法出发
- 明确不同问题的不同需求

**原则2: 数据约束意识**
- 明确数据可用性的影响
- 区分"理想情况"和"现实情况"

**原则3: 权衡透明**
- 明确每种方法的代价
- 精度 vs 计算成本 vs 数据需求

## 四、局限性深度分析框架

### 4.1 方法论层面的局限性

**L1: 前提假设问题**

| 方法类型 | 隐含假设 | 实际情况 | 失效场景 |
|----------|----------|----------|----------|
| 序列相似性方法 | 相似序列有相似相互作用 | 结构比序列更重要 | 序列相似但功能不同 |
| 结构预测方法 | 静态结构决定相互作用 | 动态构象变化重要 | IDR介导的相互作用 |
| 网络方法 | 网络拓扑信息可靠 | 网络不完整/有噪声 | 稀疏网络 |
| pLM方法 | 训练数据覆盖所有类型 | 数据偏向特定家族 | 孤儿蛋白 |

**L2: 数据依赖性问题**

```
数据偏差影响链:

实验数据偏差
    │
    ├── 强相互作用偏向 → 弱相互作用被忽略
    │
    ├── 条件特异性忽略 → 只看到"静态"PPI
    │
    ├── 物种偏向 → 人类/模式生物过度代表
    │
    └── 技术偏差 → 不同实验方法结果不一致
          │
          ▼
    训练数据偏差
          │
          ├── 模型学习偏差而非真实规律
          │
          └── 泛化能力被高估
```

**L3: 评估框架问题**

| 问题 | 表现 | 影响 |
|------|------|------|
| 训练-测试泄漏 | 同源序列同时出现 | 泛化能力高估 |
| 不平衡数据处理 | 准确率高但正样本召回率低 | 实际应用价值低 |
| 指标选择 | AUC高但实际应用效果差 | 与真实需求脱节 |
| Benchmark单一 | 只在特定数据集测试 | 方法选择困难 |

### 4.2 具体方法的失效案例分析

**案例1: AlphaFold-Multimer 的局限性**

| 局限类型 | 具体表现 | 文献证据 |
|----------|----------|----------|
| IDR界面 | pLDDT < 50 | Li 2026, Strom 2025 |
| 瞬态相互作用 | 方向预测错误 | Li 2026 |
| 膜蛋白复合物 | 螺旋位置偏差3-5Å | Li 2026 |
| 配体依赖相互作用 | 无法预测 | Li 2026 |

**案例2: 序列相似性方法的陷阱**

```
问题描述:
─────────
两个蛋白质序列相似度很高，但功能完全不同

原因分析:
─────────
1. 关键残基决定功能，而非整体相似性
2. 结构比序列更能反映相互作用
3. 进化压力作用于功能位点

文献证据:
─────────
- DeepMind团队指出ESM模型对孤儿蛋白表现差
- 多篇研究显示同源蛋白不一定保守相互作用
```

### 4.3 应用场景的局限性

| 应用场景 | 主要挑战 | 方法的局限 |
|----------|----------|------------|
| 疾病基因预测 | 网络不完整 | 遗漏关键基因 |
| 药物靶点识别 | 需要高精度 | 假阳性导致资源浪费 |
| 抗体设计 | 特异性要求 | 交叉反应预测困难 |
| 合成生物学 | 设计新相互作用 | 生成模型不可控 |
